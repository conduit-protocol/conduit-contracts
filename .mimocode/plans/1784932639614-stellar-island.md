# Plan: Cross-Chain Stream Signatures with Network Passphrase Protection

## Problem

The protocol currently only supports direct `require_auth()` for stream creation. To enable cross-chain streams (e.g., relaying stream intents from another chain or supporting gasless meta-transactions), the protocol needs off-chain signature verification. Without including the **network passphrase** (Stellar's equivalent of EVM chainId) in the signed payload, a signature valid on testnet could be replayed on mainnet or vice versa. Similarly, without a nonce, a single signature could be replayed multiple times on the same network.

## Approach

Add a `create_stream_with_signature` function to `DripFactory` that accepts an ed25519 signature over a structured message containing all stream parameters **plus** the network passphrase and a monotonically increasing nonce per sender. The signature is verified on-chain using `env.crypto().ed25519_verify()`.

The signed payload is a domain-separated hash of:
```
domain:    "Conduit" + contract_address (factory address)
struct:    sender | recipient | token | deposit | rate_per_sec | start_time | end_time | clawback | nonce | network_passphrase
```

This mirrors EIP-712's domain separation concept but uses Soroban's native ed25519 + SHA-256 primitives.

### 3. `contracts/factory/src/signature.rs` ŌĆö NEW FILE: Signature verification module

This module contains:

- **`SignaturePayload` struct** (`#[contracttype]`): All fields that get signed
  - `sender: Address`
  - `recipient: Address`
  - `token: Address`
  - `deposit: i128`
  - `rate_per_sec: i128`
  - `start_time: u64`
  - `end_time: u64`
  - `clawback: bool`
  - `nonce: u64`
  - `deadline: u64` (0 = no expiry)
  - `network_passphrase: Bytes` (the full Stellar network passphrase string)

- **`DOMAIN_SEPARATOR_PREFIX` constant**: `b"Conduit Stream Creation v1"` ŌĆö domain tag for hash separation

- **`hash_payload(env, factory_address, payload) -> BytesN<32>`** function:
  1. Compute domain hash: `SHA256(DOMAIN_SEPARATOR_PREFIX || factory_address)`
  2. Compute struct hash: `SHA256(sender || recipient || token || deposit || rate_per_sec || start_time || end_time || clawback || nonce || deadline || network_passphrase)`
  3. Return `SHA256(domain_hash || struct_hash)` ŌĆö the final digest to verify against

- **`verify_signature(env, factory_address, payload, public_key, signature) -> Result<(), Error>`** function:
  1. Hash the payload via `hash_payload`
  2. Call `env.crypto().ed25519_verify(&public_key, &digest, &signature)`
  3. Return `Ok(())` or `Err(Error::InvalidSignature)`

### 4. `contracts/factory/src/lib.rs` ŌĆö Add `create_stream_with_signature` function

New public function on `DripFactory`:

```rust
pub fn create_stream_with_signature(
    env: Env,
    sender: Address,        // recovered from public key, NOT require_auth'd
    recipient: Address,
    token: Address,
    deposit: i128,
    rate_per_sec: i128,
    start_time: u64,
    end_time: u64,
    clawback: bool,
    nonce: u64,
    deadline: u64,          // 0 = no expiry
    public_key: BytesN<32>, // ed25519 public key of the signer
    signature: BytesN<64>,  // ed25519 signature
) -> Result<u64, Error>
```

**Flow:**
1. Check emergency pause
2. Validate all stream parameters (identical to `create_stream`)
3. **Network passphrase check**: Verify `network_passphrase` in the signed payload matches `env.ledger().network_passphrase()` (prevents cross-network replay)
4. **Deadline check**: If `deadline > 0`, verify `env.ledger().timestamp() <= deadline`
5. **Nonce check**: Load sender's current nonce from `DataKey::SignatureNonce(sender)`, verify it matches the `nonce` in the payload
6. **Signature verification**: Build the `SignaturePayload`, hash it, verify via `env.crypto().ed25519_verify()`
7. **Increment nonce**: Save `nonce + 1` to `DataKey::SignatureNonce(sender)`
8. **Proceed with stream creation**: Identical to `create_stream` from this point (governor bounds, deploy, index)

## Security Properties

| Threat | Mitigation |
|--------|-----------|
| Cross-network replay (testnetŌåÆmainnet) | Network passphrase embedded in signed payload; verified against `env.ledger().network_passphrase()` |
| Same-network replay | Monotonic per-sender nonce; incremented on each use |
| Expired signature abuse | Deadline field; rejected if `deadline > 0 && now > deadline` |
| Parameter tampering | All stream parameters included in the signed hash |
| Signature forgery | Ed25519 verification via host `env.crypto().ed25519_verify()` |
| Factory address confusion | Factory contract address included in domain separator |
